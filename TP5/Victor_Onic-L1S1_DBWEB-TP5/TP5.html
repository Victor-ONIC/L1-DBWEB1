Introduction à W3.CSS

L'objectif de ce TP est de passer en revue les possibilités offerte par un framework CSS. Le principe est d'utiliser une feuille de style pré-définie (un peu comme dans les TP précédents, avec html5.css, mais avec plus de possibilités). Pour cela, parmi les plus performants (Bootstrap, Normalize, W3.CSS...) notre choix s'arrête sur W3.CSS proposé par le W3Schools : il est simple, minimaliste, et ne repose pas trop sur l'utilisation du JavaScript.

En effet la majorité des frameworks sont basés sur des combinaisons CSS+JavaScript. Nous verrons JavaScript lors du prochain semestre, aussi nous limiterons notre usage de W3.CSS à tout ce qui ne nécessite pas le recours a JavaScript pour l'instant.

Attention : Toutefois, n'hésitez pas à lire les exemples fournis avec ce language, pour commencer à vous familiariser et préparer son étude approfondie.

La documentation de W3.CSS est fournie, avec de nombreux exemples, sur le site W3Schools.

W3.CSS repose sur une organisation des pages HTML compatible avec le Responsive Design : c'est à dire que la même page va s'adapter automatiquement aux différentes conditions dans lesquelles elle peut être lue. Par exemple, entre votre écran d'ordinateur (grand) et un smartphone (petit). Pour cela le principe généralement utilisé est de diviser chaque ligne de la page en un certain nombre de colonnes (grid), par exemple 12. Ensuite le concepteur peut définir comment les objets qu'il utilise (<div> ou balises sémantiques du HTML5, <header>, <nav>...) vont se répartir sur les 12 colonnes, en fonction de la taille de la fenêtre active : par exemple un <div> ne prend que 6 colonnes en 'grand' (et donc peut partager la place avec d'autres <div> sur les 6 colonnes restantes) mais passe à 12 en 'petit' (et donc occupe tout l'espace en largeur, les autres <div> apparaitront 'à la ligne'). Mais ça n'est pas la seule façon de faire, vous pouvez en apprendre plus sur le Responsive Design sur W3Schools.

Ensuite le framework propose de nombreuses fonctionnalités permettant d'améliorer facilement le rendu visuel des pages web, et aussi de normaliser l'apparence de certaines fonctions (notions de tag, badges, modals, themes etc). La liste est longue, vous pouvez les passer en revue dans la documentation sans vous attarder sur chacun pour l'instant (vous y reviendrez au besoin).

A faire :

  • Récupérer l'archive du TP dans le lien après le sujet
  • Ouvrir la page web w3cssexemple_diff.html. Vérifier dans le code qu'elle repose bien sur l'utilisation de W3.CSS (ainsi que sur la Police de caractère Roboto, fournie par Google, et les icônes de Font Awsome, fournies par Couldfare).
  • Retrouver dans la documentation la définition des classes W3.CSS utilisées dans la page. Au passage on rappelle que le fichier CSS de W3.CSS est aussi accessible en lecture (copier-coller de l'URL de la balise link/stylesheet dans la barre d'adresse) et permet donc de vérifier quelles propriétés sont associées aux classes utilisées. Mais la documentation est plus claire au départ...
  • Ensuite ouvrir la page dans votre navigateur, puis vérifier ses propriétés Responsive Design. Passer de écran large à smartphone et vérifier le placement des 2 colonnes selon les cas :
    • soit en redimensionnant (à la main) votre navigateur,
    • soit en activant le mode "Responsive" de votre navigateur (en général dans le menu des outils développement).
  1. Le premier exercice est celui des "7 différences" : 
    1. ouvrir la page web fournie dans l'archive et trouver les 7 différences avec la page donnée ci-dessous
    2. réaliser les modifications pour reproduire exactement la page d'exemple
    3. vérifier à nouveau le comportement responsive après vos changements
  2. Dans la page, bien qu'homogènes, les couleurs sont définies pour chaque élément, passer la page en version thème : déclarer un thème W3.CSS global puis ensuite définir pour chaque élément de la page la couleur RELATIVE au thème qu'il doit prendre. Vérifier que vous pouvez changer de thème rapidement (en modifiant juste son nom) tout en gardant un jeu de couleurs cohérent au sein de votre page.
  3. Introduire un nouveau bloc (card) avant 'Formation' dans la seconde colonne. Insérer un tableau (pré-formaté par W3.CSS, à vous de choisir le style adapté parmi les différentes options) contenant 2 lignes ('Ma Formation', 'Mon Expérience') et 3 colonnes (pour les périodes temporelles avant 2010, 2010-13, 2013-15). Mettre dans les cellules des liens vers les parties correspondantes du CV.

Aide : les différences entre la page de l'image et le fichier fourni portent sur :

  1. le rapport de taille des colonnes (il doit être 1/3 2/3),
  2. les icônes du CV ne sont pas toutes identiques (et la première tourne sur elle-même),
  3. les titres à l'intérieur des blocs 'Formation' et 'Expérience', ils sont légèrement "opacifiés" dans l'image (pas forcement facile à voir d'ailleurs !!),
  4. la photo prend toute la largeur de la colonne,
  5. le nom n'est pas au milieu,
  6. pour le stage chez maboite le bouton "current" apparait comme un tag au bords arrondis (pas comme un texte normal),
  7. il manque les icônes des réseaux sociaux.
