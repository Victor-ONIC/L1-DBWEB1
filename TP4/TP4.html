Introduction à HTML 5

Le HTML version 5, introduit ces dernières années par le W3C, présente un certain nombre de nouveautés très intéressantes pour la conception internet. Principalement de nouveaux éléments/balises, mais aussi de nouvelles API (Application Programming Interface). Ces dernières seront vues plus tard durant votre formation. L'objectif de ce TP est de passer en revue les principales nouvelles balises de HTML 5. En particulier, nous commencerons par celles permettant la structuration des documents web, avant d'aborder un nouvel élément permettant la réalisation de dessins en HTML.

Attention : l'introduction de nouvelles balises va de pair avec la disparition d'autres (se référer à un site de référence pour connaitre la liste des balises devenues obsolètes).

1. Structuration de documents web

Dans nos premiers TP, une page web était constituée d'un texte simple, de haut en bas de la fenêtre active. Mais, en général une page web est composée de plusieurs éléments organisés de façon spatiale. Par exemple : un bandeau de titre en haut sur toute la largeur, un menu à gauche sur 1/4 de la largeur, une fenêtre de contenu principal sur le reste de la largeur et, enfin, un pied de page sur toute la largeur. Mais bien sur une infinité de combinaisons sont possibles.

La structure d'une page web peut être définie en HTML de trois manières principales :

  • un tableau : on utilise les balises <table>/<tr>/<td>, vues au TP2, pour organiser le contenu de la page, 
  • des cadres, ou frames : technique obsolète, mais peut encore en rencontrer sur le web,
  • une combinaison blocs HTML (<div>) et CSS : on regroupe les différents contenus de la page dans des balises <div>, chacune avec un identifiant unique qui permet de lui associer des règles CSS spécifiques pour son positionnement (et sa mise en forme en général, comme vu au TP3).

Nous allons mettre en oeuvre la dernière technique en utilisant les nouvelles balises HTML 5 :

  • <article>
  • <aside>
  • <details>
  • <figcaption>
  • <figure>
  • <footer>
  • <header>
  • <main>
  • <mark>
  • <nav>
  • <section>
  • <summary>
  • <time>

On dispose ainsi de "blocs" de contenu avec l'information sur le "type" de contenu (ce que ne permet pas l'utilisation des <div>). Pour débuter nous utiliserons un fichier CSS fourni, donné dans le sujet plus loin (html5.css).

A faire :

1- Créer un document web simple en HTML 5, avec uniquement des éléments de structuration et sans éléments de mise en forme, comprenant :

  • un entête contenant un titre et sous-titre pour la page
  • un menu de navigation : il s'agit d'un bloc contenant une liste (non ordonnée) de liens vers d'autres pages (ne pas indiquer de vrais liens ici, juste mettre 'href="#"')
  • un contenu composé de 3 sections comprenant chacune 2 articles. Chaque article est constitué d'un titre et d'un paragraphe de contenu
  • un des articles contient un paragraphe de détails (que le lecteur choisit de lire ou pas), avec une phrase 'résumé' (qui apparait tout le temps)
  • un pied de page reprenant quelques informations générales sur la page, comme l'auteur et la date.

Tous les éléments demandés ci-dessus peuvent être réalisée avec une balise du HTML 5 bien précise à trouver.

Pour le contenu de tous les éléments vous pouvez écrire ce que vous voulez, ou aller chercher du contenu sur internet

2- Insérer une balise <link> dans l'entête <head> de votre fichier pour lui associer le fichier CSS html5.css

3- Vérifier la bonne mise en page de votre fichier après l'association avec le fichier CSS

4- Modifier le fichier CSS pour permettre au menu de navigation de se trouver à droite du contenu sur toute la hauteur. Pour cela :

  1. Déclarer le bloc menu comme flottant à droite et de largeur 20%
  2. Repasser les items (<li>) en mode d'affichage en block (sinon ils sont "collés" les uns les autres)
  3. Enfin, pour séparer le menu des sections :
    • Regrouper les sections dans un seul container (main, par exemple)
    • Demander que l'overflow du container soit caché : cela impose que la boîte d'affichage du main contienne exactement son contenu, sans l'autoriser à déborder sur les autres boîtes

2. Réalisation de dessins web

Plusieurs balises permettent d'insérer des images dans un document web. <img> permet l'affichage d'une image donnée dans un fichier séparé, <canvas> et <svg> permettent de définir l'image directement à partir d'instructions HTML. <canvas> ayant recours à des instructions JavaScript, nous nous limiterons à <svg> pour l'instant.

A faire :

1- Ajouter une balise <svg> à la suite de la page précédente, dans un article.

2- Intégrer le dessin suivant à la fin de votre fichier comme logo dans le pied de page et modifier le parcours du texte CERI pour le faire partir du coin haut-droit du dessin et arriver dans la gueule du crocodile :
